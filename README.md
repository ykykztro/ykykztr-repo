# WindowQuickJump

窗口 / 标签页编号直达工具（AutoHotkey v2）。把常用窗口或标签页「钉」到数字键 1\~9 上，一键跳回，不用再在任务栏里翻找。

> 快捷键统一使用**左 Alt**（右 Alt 不会触发），避免和右 Alt 组合的快捷键冲突。

## 它能做什么

- **9 个编号槽位（Slot 1\~9）**：把任意窗口或浏览器标签页、VS Code 窗口绑定到数字键。
- **浏览器标签页精确定位**：跳回浏览器时，会自动定位到当初绑定的**那一页标签页**——无论你后来开了多少个、关了多少个、还是把它挪到了别的位置。
- **VS Code 窗口直达**：跳回 VS Code 时，直接切到绑定的那个 VS Code 窗口（不区分内部标签页）。
- **右下角轻提示**：所有操作反馈显示在屏幕**右下角**，弱存在感、不挡视线，且切回视频页后**空格仍能控制播放**。
- **IME 友好**：切回页面后焦点留在页面内容，微软拼音候选框正常跟随光标，不会被钉到左上角。
- **自动管理员提权**：部分窗口需要管理员权限才能激活，脚本启动时自动请求。
- **失效检测**：绑定的窗口若已关闭，自动清空该槽位并提示。

## 快捷键

| 快捷键 | 作用 |
|--------|------|
| `左Alt + 1`\~`左Alt + 9` | **智能切换**：槽位为空 → 绑定当前窗口；槽位已有内容 → 跳转到它 |
| `Ctrl + 左Alt + 1`\~`Ctrl + 左Alt + 9` | **强制覆盖**：用当前窗口覆盖该槽位，无论是否已有绑定 |

> 槽位状态是**内存态**：脚本退出或重启后清空，需重新绑定。

## 使用流程

1. 切到你常驻的窗口（如 VS Code），按 `左Alt + 1` → 提示「Slot 1 已绑定: xxx」。
2. 在浏览器里打开想记住的标签页（比如正在播放视频的那个），按 `左Alt + 2` → 提示「Slot 2 已绑定: 页面标题」。
3. 之后在任意窗口按 `左Alt + 1` 立刻跳回 VS Code；按 `左Alt + 2` 跳回浏览器并自动定位到那个标签页。
4. 想替换某个槽位的绑定，按对应 `Ctrl + 左Alt + N` 强制覆盖即可。

## 浏览器标签页是怎么定位的（原理）

绑定那一刻，脚本会记下**干净的标签页名字**（自动去除浏览器后缀、Edge 多标签计数、profile 名、音视频播放符号等）。跳转时：

1. 如果当前已经是目标页 → 直接提示，不切换。
2. 否则用**顺序循环遍历**：先正向 `Ctrl + PageDown` 逐标签查找，遇到已见过的标题即表示绕回一圈；若浏览器不回环且目标在左侧，再反向 `Ctrl + PageUp` 兜底。整个过程基于**清洗后的干净标题**做匹配，因此不受标签数量、顺序、新建标签页影响。

支持的浏览器：Chrome、Edge 等 Chromium 内核浏览器（窗口类名 `Chrome_WidgetWin_1` / `Chrome_WidgetWin_0`）。

## VS Code 是怎么定位的（原理）

VS Code 与 Chrome 同属 `Chrome_WidgetWin_1` 窗口类，脚本**先用进程名 `Code.exe` 区分**，避免被误判为浏览器。跳转时直接 `WinActivate` 切到该 VS Code 窗口即算成功（不做窗口内的文件级定位）。

## 焦点行为

跳回浏览器标签页后，脚本仅通过 `WinActivate` + 键盘 `Ctrl + PageDown` 循环切标签来定位，**不再用 `ControlFocus` 强行聚焦渲染控件**——这样切回正在播放视频的标签页后，直接按空格就能暂停 / 播放，同时微软拼音候选框也能正常跟随光标（早期版本因 `ControlFocus` 到无光标渲染控件而导致候选框被钉在左上角，现已修复）。

## 限制

- 不允许绑定 AHK 自身窗口（会提示「不能绑定此窗口」）。
- 同一浏览器窗口内若有**多个同名标签页**，顺序遍历会停在第一个匹配项。
- 槽位数据保存在内存，脚本重启后需重新绑定。

## 安装与启动

1. 安装 [AutoHotkey v2](https://www.autohotkey.com/)（必须是 v2，非 v1）。
2. 双击 `WindowQuickJump.ahk` 运行。
3. 屏幕右下角出现启动提示「WindowQuickJump 已启动」，之后常驻后台。
