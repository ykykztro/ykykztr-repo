# WindowQuickJump

窗口 / 标签页编号直达工具（AutoHotkey v2）。把常用窗口或标签页「钉」到数字键 1\~9 上，一键跳回，不用再在任务栏里翻找。

## 它能做什么

- **9 个编号槽位（Slot 1\~9）**：把任意窗口或浏览器标签页、VS Code 编辑器绑定到数字键。
- **浏览器标签页精确定位**：跳回浏览器时，会自动定位到当初绑定的**那一页标签页**——无论你后来开了多少个、关了多少个、还是把它挪到了别的位置。
- **VS Code 文件直达**：跳回 VS Code 时，按文件名直达绑定的那个文件（不受分屏、多编辑器组影响）。
- **右下角轻提示**：所有操作反馈显示在屏幕**右下角**，弱存在感、不挡视线，且切回视频页后**空格仍能控制播放**。
- **自动管理员提权**：部分窗口需要管理员权限才能激活，脚本启动时自动请求。
- **失效检测**：绑定的窗口若已关闭，自动清空该槽位并提示。

## 快捷键

| 快捷键 | 作用 |
|--------|------|
| `Alt + 1`\~`Alt + 9` | **智能切换**：槽位为空 → 绑定当前窗口；槽位已有内容 → 跳转到它 |
| `Ctrl + Alt + 1`\~`Ctrl + Alt + 9` | **强制覆盖**：用当前窗口覆盖该槽位，无论是否已有绑定 |

> 槽位状态是**内存态**：脚本退出或重启后清空，需重新绑定。

## 使用流程

1. 切到你常驻的窗口（如 VS Code），按 `Alt + 1` → 提示「Slot 1 已绑定: xxx」。
2. 在浏览器里打开想记住的标签页（比如正在播放视频的那个），按 `Alt + 2` → 提示「Slot 2 已绑定: 页面标题」。
3. 之后在任意窗口按 `Alt + 1` 立刻跳回 VS Code；按 `Alt + 2` 跳回浏览器并自动定位到那个标签页。
4. 想替换某个槽位的绑定，按对应 `Ctrl + Alt + N` 强制覆盖即可。

## 浏览器标签页是怎么定位的（原理）

绑定那一刻，脚本会记下**干净的标签页名字**（自动去除浏览器后缀、Edge 多标签计数、profile 名、音视频播放符号等）。跳转时：

1. 如果当前已经是目标页 → 直接提示，不切换。
2. 否则用浏览器自带的 **标签搜索面板（`Ctrl + Shift + A`）**，直接按名字搜索并回车跳转 —— 这种方式**不受标签数量、顺序、新建标签页、睡眠标签页影响**。
3. 若标签搜索面板不可用或未命中，再退回到**顺序遍历**（`Ctrl + PageDown` 正向 + `Ctrl + PageUp` 反向兜底），确保仍能找回。

支持的浏览器：Chrome、Edge 等 Chromium 内核浏览器（窗口类名 `Chrome_WidgetWin_1` / `Chrome_WidgetWin_0`）。

> ⚠️ 标签搜索依赖 `Ctrl + Shift + A` 这个内置快捷键，需 Edge/Chrome 较新版本（约 120+）。若你的版本没有该功能，会自动走顺序遍历兜底。

## VS Code 是怎么定位的（原理）

VS Code 与 Chrome 同属 `Chrome_WidgetWin_1` 窗口类，脚本**先用进程名 `Code.exe` 区分**，避免被误判为浏览器。跳转时通过 **`Ctrl + P` 快速打开**按文件名直达绑定的文件，不依赖标签页方向键，因此不受「最近使用顺序」、分屏、多编辑器组的影响。

## 焦点行为

跳回浏览器标签页后，脚本会把键盘焦点**送回网页渲染区域**（而非地址栏 / ⋮ 菜单 / 标签栏）。这样切回正在播放视频的标签页后，直接按空格就能暂停 / 播放。

## 限制

- 不允许绑定 AHK 自身窗口（会提示「不能绑定此窗口」）。
- 同一浏览器窗口内若有**多个同名标签页**，标签搜索会取搜索结果的第一项。
- 槽位数据保存在内存，脚本重启后需重新绑定。

## 安装与启动

1. 安装 [AutoHotkey v2](https://www.autohotkey.com/)（必须是 v2，非 v1）。
2. 双击 `WindowQuickJump.ahk` 运行。
3. 屏幕右下角出现启动提示「WindowQuickJump 已启动」，之后常驻后台。
